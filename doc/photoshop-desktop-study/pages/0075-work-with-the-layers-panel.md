# Work with the Layers Panel

- Entry Type: Core interface workflow
- Category: Create and manage layers > Get started with layers
- Source: https://helpx.adobe.com/photoshop/desktop/create-manage-layers/get-started-layers/work-with-the-layers-panel.html

**Function Summary**
The Layers panel is the main control surface for managing a Photoshop document's layer stack. It is where layers are selected, hidden, reordered, renamed, and created.

**What This Function Does**
- Displays the complete layer stack for the active document
- Lets users show, hide, select, and reorder layers
- Provides direct access to basic layer creation and naming actions

**Where You Find It**
- `Window > Layers`

**Primary Interface Elements**
- Layers panel
- layer row
- eye visibility icon
- layer name
- layer thumbnail
- `Create a new layer` button

**How To Use It**
1. Open the panel with `Window > Layers` if it is not already visible.
2. Click a layer to make it active.
3. Click the eye icon beside a layer thumbnail to show or hide it.
4. Drag a layer up or down in the stack to change its order.
5. Double-click a layer name, type the new name, and press `Enter` on Windows or `Return` on macOS to rename it.
6. Click `Create a new layer` to add a blank layer.

**Mouse, Keyboard, and Menu Input**
- `Mouse`
  Click layers, eye icons, layer names, and drag layers to new positions.
- `Menus`
  Use `Window > Layers` to reveal the panel.
- `Keyboard`
  Press `Enter` or `Return` after renaming a layer.

**User Interface Behavior**
- The panel updates to reflect the active document's layer structure.
- Hidden layers remain in the file and can be re-shown instantly.
- Reordering changes stacking order visually on the canvas but does not merge the layers.

**Expected Result**
- The user can control the structure and visibility of document elements directly from one panel.

**Practical Use Cases**
- Reorder a logo above a background image.
- Hide guide or reference layers temporarily.
- Create and name layers cleanly while building a composition.
