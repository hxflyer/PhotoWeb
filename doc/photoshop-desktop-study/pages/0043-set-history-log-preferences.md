# Set History Log Preferences

- Entry Type: Edit auditing preference
- Category: Get started > Set up toolbars and panels
- Source: https://helpx.adobe.com/photoshop/desktop/get-started/set-up-toolbars-panels/history-log-preferences.html

**Function Summary**
History Log records Photoshop editing activity for documentation purposes. It can store the log in file metadata, in an external text file, or in both places, and it lets you choose how much detail should be recorded.

**What This Function Does**
- Tracks editing activity for documentation and review
- Stores logs in metadata, a text file, or both
- Lets you choose between session-only, concise, and detailed logging

**Where You Find It**
- `Edit > Preferences > History & Content Credentials` on Windows
- `Photoshop > Preferences > History & Content Credentials` on macOS

**Primary Interface Elements**
- `History Log`
- storage target options: `Metadata`, `Text File`, `Both`
- `Choose` file location button
- `Edit Log Items`
- `Sessions Only`
- `Concise`
- `Detailed`

**How To Use It**
1. Open `History & Content Credentials` preferences.
2. Enable the `History Log` checkbox.
3. Choose where Photoshop should save the log: `Metadata`, `Text File`, or `Both`.
4. If you choose `Text File` or `Both`, click `Choose` and define the file path and filename.
5. Set the logging detail level from `Edit Log Items`.
6. Confirm with `OK`.

**Log Detail Levels**
- `Sessions Only`
  Records when Photoshop sessions start and end
- `Concise`
  Adds history-style edit entries along with session information
- `Detailed`
  Includes more complete action-level logging and can produce larger logs

**Mouse, Keyboard, and Menu Input**
- `Mouse`
  Use the preferences controls to enable logging, choose the storage destination, and set the detail level.
- `Menus`
  Open the `History & Content Credentials` preferences from the application menu.
- `Keyboard`
  No dedicated default hotkey is described for this preference screen.

**User Interface Behavior**
- Logging runs automatically after it is enabled.
- Metadata logging embeds the history record in the file when supported.
- Text-file logging stores the record separately at the chosen path.
- Detailed logging may increase file size or affect performance more than lighter logging modes.

**Expected Result**
- Editing history becomes easier to audit for workflow review, client documentation, or internal records.

**Practical Use Cases**
- Keep a documented edit trail for production or legal review.
- Store a separate text log for batch or archival workflows.
- Use concise logs when you want traceability without maximum detail overhead.
