# View History Logs

- Entry Type: Edit audit review
- Category: Get started > Set up toolbars and panels
- Source: https://helpx.adobe.com/photoshop/desktop/get-started/set-up-toolbars-panels/view-history-logs.html

**Function Summary**
This function explains where to review the edit logs produced by Photoshop History Log. Logs can be read either from embedded file metadata or from a separately saved text file.

**What This Function Does**
- Opens metadata-based history information inside Photoshop
- Points users to text-file logs saved outside the document
- Supports review, workflow tracing, and documentation checks

**Where You Find It**
- `File > File Info > Photoshop`
- External log file path chosen in History Log preferences

**Primary Interface Elements**
- `File`
- `File Info`
- `Photoshop` tab
- `History` section
- external text log file

**How To Use It**
1. Open the Photoshop file whose log you want to inspect.
2. Go to `File > File Info`.
3. Open the `Photoshop` tab.
4. Locate the `History` section to review metadata-based log entries.
5. If the log was configured as a text file, open the saved log file in any text editor at the path you previously chose.

**Mouse, Keyboard, and Menu Input**
- `Mouse`
  Use the File Info dialog and click into the `Photoshop` tab.
- `Menus`
  Use `File > File Info` for metadata logs.
- `Keyboard`
  No dedicated default hotkey is described for viewing history logs.

**User Interface Behavior**
- Metadata-based logs stay associated with the file when saved in supported formats and workflows.
- Text-file logs are stored separately and reviewed outside Photoshop.
- What you see depends on the History Log detail level selected during logging setup.

**Expected Result**
- You can inspect what Photoshop recorded about the edit session and determine whether the log contains enough information for your needs.

**Practical Use Cases**
- Verify whether a client-deliverable file carries an embedded edit record.
- Review a text log after a production session.
- Confirm that History Log settings are working as intended.
